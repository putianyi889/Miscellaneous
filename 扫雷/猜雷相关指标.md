# 猜雷相关指标

## 胜率相关的`Luck`和`pLuck`
> Winrate-related `Luck` and `pLuck`

每次操作都附带一个安全的概率。所有操作的安全概率相乘得到整局胜率，`Luck`。在高难自定义，或者长连胜中，`Luck`往往非常小，因此用`pLuck = -log10(Luck)`尺度更合适。
> Every action is associated with a safety probability. All such probabilities multiply together to get the winrate of the game, named `Luck`. In high-difficulty or long winstreak, `Luck` tends to be very small, so `pLuck = -log10(Luck)` is a better indicator.

## 开速相关
> Endurance related

考虑每个bv解决的时间点的`Luck`，记为 $`\text{Luck}_1, \dots, \text{Luck}_{\text{Bv}}`$
> Consider the `Luck` value at the timestamp when each Bv is solved.

$$\text{Expected Bv Solved Per Game} = \sum_{n=1}^{\text{Bv}}\text{Luck}_n $$

$$\text{Expected Bv Solved Per Win} = \frac{\text{Expected Bv Solved Per Game}}{\text{Luck}}$$

考虑每个bv解决的时间点的`Time`，记为 $`\text{Time}_1, \dots, \text{Time}_{\text{Bv}}`$
> Consider the `Time` value at the timestamp when each Bv is solved.

$$\text{Expected Time Per Game} = \sum_{n=1}^{\text{Bv}}\text{Luck}_n(\text{Time}_n-\text{Time}_{n-1})$$

$$\text{Expected Time Per Win} = \frac{\text{Expected Time Per Game}}{\text{Luck}}$$

`Expected Time Per Win`就是开速决定性的指标。
> `Expected Time Per Win` is the decisive indicator for endurance score.

## Remarks
- 猜雷不一定会推进Bv
  > Guessing doesn't always progress Bv.
- 单次操作解决多个Bv，公式里应该理解为多个加项
  > A single action can solve multiple Bv, which correspond to multiple terms.
- 连续无猜解决多个Bv，公式里体现为多项相同Luck相加，尽管算法可以优化
  > Consecutive no-guessing actions can solve multiple Bv, which correspond to multiple terms having the same `Luck` value.
- 以上所有指标都可以在局中更新。
  > All the indicators can be updated in game (not while playing, but during replay).
